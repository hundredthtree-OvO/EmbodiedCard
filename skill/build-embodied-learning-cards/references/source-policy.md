# Source policy

## Priority

1. Original paper or peer-reviewed publication.
2. Official project page and official implementation.
3. Official framework documentation.
4. University textbook, course notes, or author tutorial.
5. High-quality technical exposition used only to improve intuition.

## Rules

- Browse when a named paper, repository, current architecture, benchmark result, API, or implementation detail is involved.
- Use primary sources for technical claims whenever available.
- Do not cite a search-results page.
- Link directly to the source in the card's reference section.
- Separate a source-stated fact from an inference made by the card author.
- Cross-check equations, signs, index direction, tensor shapes, training/inference differences, and reported hyperparameters.
- Do not turn an illustrative convention into a universal rule. For example, reconstruction loss depends on the likelihood model.
- Avoid long quotations; paraphrase and explain.
